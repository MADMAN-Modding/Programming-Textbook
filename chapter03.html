Chapter 3

Conditionals

if / else, booleans, logical operators, and a bouncing ball.

Alright, alright, alright! In Chapter 2 we made things move. But our shapes were dumb — they just kept going, right off the edge of the canvas, never to be seen again. In this chapter we're going to give them a brain. A simple one. We're going to teach them how to make a decision. That's what conditionals are for. Let's go!

The if statement #

A conditional says: "If this thing is true, run this code. Otherwise, skip it." Here's the basic shape:

if (something is true) {
  // run this code
}

For example, "if the mouse is on the right half of the canvas, paint the background red":

function draw() {
  if (mouseX > 200) {
    background(255, 0, 0);
  }
}

Coding Train 3.1 — Conditional Statements

Comparison operators #

The thing inside the parentheses needs to be something that JavaScript can answer with yes or no. These operators are the questions you can ask:

OperatorMeaningExample
>greater thanmouseX > 200
<less thany < height
>=greater than or equal tox >= 100
<=less than or equal toscore <= 0
==equal to (loose)x == 10
===equal to (strict)x === 10
!= / !==not equal tox !== 0
Prefer === over == JavaScript's == tries to convert types before comparing, which leads to weirdness ("5" == 5 is true!). The triple equals === demands they be exactly the same. Use === unless you have a really good reason not to.

else and else if #

What if you want code to run when the condition is not true? Add an else:

if (mouseX > 200) {
  background(255, 0, 0);    // right side: red
} else {
  background(0, 0, 255);    // left side: blue
}

Need more than two branches? Chain them with else if:

if (mouseX < 133) {
  background(255, 0, 0);
} else if (mouseX < 266) {
  background(0, 255, 0);
} else {
  background(0, 0, 255);
}

p5 checks them top-to-bottom and runs only the first one that's true. As soon as one matches, it skips the rest.

Coding Train 3.2 — The Bouncing Ball

The bouncing ball #

Here's the canonical example. We have a ball with a position x and a speed xspeed. Every frame, we add the speed to the position. When the ball hits the edge of the canvas, we reverse the speed. That's bouncing.

let x = 200;
let y = 200;
let xspeed = 3;
let yspeed = 2;

function setup() {
  createCanvas(400, 400);
}

function draw() {
  background(220);
  fill(237, 34, 93);
  ellipse(x, y, 40, 40);

  x += xspeed;
  y += yspeed;

  // bounce off the right or left wall
  if (x > width || x < 0) {
    xspeed = -xspeed;
  }
  // bounce off the top or bottom
  if (y > height || y < 0) {
    yspeed = -yspeed;
  }
}

Read that xspeed = -xspeed; line. If xspeed was 3, now it's -3. The ball was moving right; now it moves left. That single line is the entire trick.

The radius problem This version checks the center of the ball against the wall, so half the ball pokes through before bouncing. Fix: use the ball's radius. Try if (x > width - 20 || x < 20) for a 40-pixel-wide ball.

Logical operators: && and || #

You can glue multiple conditions together:

OperatorMeansTrue when
&&andboth sides are true
||orat least one side is true
!notflips true to false and vice versa
 
// inside a 100x100 box at the top-left?
if (mouseX < 100 && mouseY < 100) {
  fill(0, 255, 0);
}

// mouse off either edge?
if (mouseX < 0 || mouseX > width) {
  print("out of bounds!");
}

Coding Train 3.3 — Logical Operators

Booleans #

A boolean is a variable that holds only true or false. Useful for tracking state — is the game paused? Has the player won yet? Is the door open?

let isPaused = false;

function draw() {
  if (isPaused) {
    return;   // skip the rest of draw()
  }
  // ... rest of the sketch
}

function mousePressed() {
  isPaused = !isPaused;   // flip it: true ↔ false
}

That last line is a toggle. Press the mouse, pause; press again, unpause. The ! means "not," so !isPaused reads as "the opposite of whatever isPaused currently is."

Coding Train 3.4 — Boolean Variables

Common Beginner Mistakes #

Using = instead of === if (score = 100) doesn't check the score, it sets it to 100, and the condition is always truthy. You wanted if (score === 100).
Missing curly braces Always use { }, even for one-line ifs. Beginners get bitten when they add a second line and forget that without braces, only the first line is part of the if.
"or" written as "or" JavaScript doesn't speak English. and, or, and not are &&, ||, and !. (The pipe character | is on the same key as \ on most keyboards.)
Comparing the wrong way if (0 < x < 100) looks like math but isn't. JavaScript evaluates 0 < x first (giving true or false), then compares that to 100, which is nonsense. Write it as if (x > 0 && x < 100).
