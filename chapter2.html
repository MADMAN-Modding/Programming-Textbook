Chapter 2: Variables

2.1 What Are Variables? Making Your Code Remember Things!

Okay, okay, so you've made some beautiful static drawings with shapes and colors. That's amazing! But what if I told you there's a way to make your sketches come ALIVE? What if your circles could follow your mouse? What if things could move and change and remember information? Enter: VARIABLES!

Think of a variable like a box with a label on it. You can:

  • Put things in the box (store values)
  • Look at what's in the box (read values)
  • Change what's in the box (update values)
  • The label tells you what kind of stuff is in the box (the variable name)

In programming, that "stuff" is data - numbers, words, colors, positions, you name it!

Your First Variables: mouseX and mouseY

Before we learn how to make our own variables, let's play with some that p5.js gives us for FREE! The most fun ones are mouseX and mouseY.

function setup() {
  createCanvas(400, 400);
}

function draw() {
  background(220);
  circle(mouseX, mouseY, 50);
}

WHOA! The circle follows your mouse! How does this magic work?

  • mouseX is a variable that always contains the x position of your mouse
  • mouseY is a variable that always contains the y position of your mouse
  • p5.js updates these values automatically 60 times per second!

Let's see what these values actually are:

function setup() {
  createCanvas(400, 400);
}

function draw() {
  background(220);
  
  // Draw circle at mouse position
  circle(mouseX, mouseY, 50);
  
  // Show the values
  fill(0);
  text("mouseX: " + mouseX, 10, 20);
  text("mouseY: " + mouseY, 10, 40);
}

Move your mouse around and watch the numbers change! These variables are constantly updating with new values.

Making a Paint Program!

Here's something cool - what if we DON'T clear the background each frame?

function setup() {
  createCanvas(400, 400);
  background(220);  // Clear background only once!
}

function draw() {
  // No background() here!
  circle(mouseX, mouseY, 20);
}

You're painting with circles! The old circles stay on the screen because we're not clearing them with background().

Other Built-in Variables

p5.js gives us lots of useful variables for free:

function setup() {
  createCanvas(400, 400);
}

function draw() {
  background(220);
  
  // width and height give us the canvas size
  circle(width/2, height/2, 100);  // Always centered!
  
  // frameCount counts how many frames have passed
  fill(0);
  text("Frame: " + frameCount, 10, 20);
}
  • width - the width of your canvas
  • height - the height of your canvas
  • frameCount - how many times draw() has run
  • mouseX - x position of mouse
  • mouseY - y position of mouse

2.2 Creating Your Own Variables

Built-in variables are great, but the REAL power comes from making your own variables! Let's learn how!

Declaring Variables with let

To create a variable in JavaScript, we use the keyword let:

let circleX = 200;
let circleY = 200;

function setup() {
  createCanvas(400, 400);
}

function draw() {
  background(220);
  circle(circleX, circleY, 50);
}

Let's break this down:

  • let - This keyword tells JavaScript "I want to create a variable!"
  • circleX - This is the name we chose for our variable (the label on the box)
  • = - This means "put the following value into the variable"
  • 200 - This is the initial value we're storing

But wait! This seems exactly the same as just writing circle(200, 200, 50). Why bother with variables?

Variables Can Change!

Here's where it gets exciting - variables can CHANGE their values:

let circleX = 0;

function setup() {
  createCanvas(400, 400);
}

function draw() {
  background(220);
  circle(circleX, 200, 50);
  
  circleX = circleX + 1;  // Move the circle to the right!
}

Your circle is MOVING! Let's understand what's happening:

  1. We start with circleX = 0
  2. We draw a circle at position (0, 200)
  3. We add 1 to circleX, so now circleX = 1
  4. Next frame: circle draws at (1, 200)
  5. We add 1 again, so circleX = 2
  6. This repeats 60 times per second!

Variable Names - Choose Wisely!

You can name your variables almost anything you want, but there are rules:

Good variable names:
  • circleX - Descriptive!
  • playerScore - Clear what it's for
  • backgroundColor - Self-explanatory
Bad variable names:
  • x - Too vague (unless it's obviously an x coordinate)
  • thing - What thing?!
  • asdfghjkl - Just... no
Invalid names (will cause errors):
  • 2fast2furious - Can't start with a number
  • my-variable - No hyphens allowed
  • let - Can't use reserved keywords

Variable Scope - Where Variables Live

WHERE you create a variable matters! Variables created outside any function are "global" - they can be used anywhere:

let globalX = 100;  // This is a global variable

function setup() {
  createCanvas(400, 400);
  let setupVariable = 50;  // This only exists in setup()
}

function draw() {
  background(220);
  circle(globalX, 200, 50);  // This works!
  // circle(setupVariable, 200, 50);  // This would cause an error!
}
Important: Variables declared inside a function only exist within that function. Variables declared outside any function can be used anywhere in your program.

2.3 Incrementing Variables - Making Things Move!

We've seen how to make things move by changing variables. Let's explore different ways to update variable values!

The Basic Way

let x = 0;

function setup() {
  createCanvas(400, 400);
}

function draw() {
  background(220);
  circle(x, 200, 50);
  
  x = x + 1;  // Add 1 to x
}

Shorthand Operators - Less Typing!

Programmers are lazy (in a good way!). We don't like typing more than necessary. So we have shortcuts:

// These all do the same thing:
x = x + 1;    // The long way
x += 1;       // Add 1 to x
x++;          // Add 1 to x (super short!)

// For subtraction:
x = x - 1;    // The long way
x -= 1;       // Subtract 1 from x
x--;          // Subtract 1 from x

// You can use += with any number:
x += 5;       // Same as x = x + 5
x -= 3;       // Same as x = x - 3
x *= 2;       // Same as x = x * 2
x /= 2;       // Same as x = x / 2

Different Speeds

Want to make things move faster or slower? Just change how much you add!

let slowX = 0;
let mediumX = 0;
let fastX = 0;

function setup() {
  createCanvas(400, 400);
}

function draw() {
  background(220);
  
  // Slow circle
  fill(255, 0, 0);
  circle(slowX, 100, 30);
  slowX += 0.5;  // Moves slowly
  
  // Medium circle
  fill(0, 255, 0);
  circle(mediumX, 200, 30);
  mediumX += 2;   // Moves at medium speed
  
  // Fast circle
  fill(0, 0, 255);
  circle(fastX, 300, 30);
  fastX += 5;     // Zooms across!
}

Going Backwards

Who says things can only move right? Let's go backwards!

let x = 400;  // Start at the right

function setup() {
  createCanvas(400, 400);
}

function draw() {
  background(220);
  circle(x, 200, 50);
  
  x = x - 2;  // Move left!
}

2.4 The random() Function - Embrace the Chaos!

Sometimes you don't want things to be predictable. Sometimes you want RANDOMNESS! The random() function is here to add some chaos to your life!

Basic Random Numbers

function setup() {
  createCanvas(400, 400);
}

function draw() {
  background(220);
  
  let x = random(0, width);   // Random x between 0 and canvas width
  let y = random(0, height);  // Random y between 0 and canvas height
  
  circle(x, y, 50);
}

SEIZURE WARNING! Just kidding! But seriously, that's a lot of circles jumping around! Every frame, we get new random positions.

How random() Works

The random() function can be used in different ways:

// Random number between 0 and 10
let r1 = random(10);

// Random number between 5 and 10
let r2 = random(5, 10);

// Random number between -5 and 5
let r3 = random(-5, 5);

Random Colors!

function setup() {
  createCanvas(400, 400);
  background(220);
}

function draw() {
  // Random color values
  let r = random(0, 255);
  let g = random(0, 255);
  let b = random(0, 255);
  
  fill(r, g, b);
  noStroke();
  
  // Random position and size
  let x = random(width);
  let y = random(height);
  let size = random(10, 50);
  
  circle(x, y, size);
}

This creates a beautiful, chaotic splatter of colorful circles!

Random Movement

Instead of moving in straight lines, let's add some random wiggle:

let x = 200;
let y = 200;

function setup() {
  createCanvas(400, 400);
  background(220);
}

function draw() {
  // Semi-transparent background for trail effect
  background(220, 220, 220, 25);
  
  // Random walk
  x += random(-3, 3);
  y += random(-3, 3);
  
  fill(0);
  circle(x, y, 20);
}

This creates a "random walker" - it wanders around randomly!

2.5 The map() Function - Mathematical Magic!

Okay, this one's a bit mind-bending, but SO useful! The map() function takes a number from one range and maps it to another range. It's like a translator for numbers!

The Problem map() Solves

Let's say you want the mouse position to control the size of a circle. But mouseX goes from 0 to 400 (the width of our canvas), and we want our circle size to go from 10 to 100. How do we convert?

This is where map() comes in!

function setup() {
  createCanvas(400, 400);
}

function draw() {
  background(220);
  
  // Map mouseX (0-400) to circle size (10-100)
  let circleSize = map(mouseX, 0, width, 10, 100);
  
  circle(200, 200, circleSize);
  
  // Show the values
  fill(0);
  text("mouseX: " + mouseX, 10, 20);
  text("size: " + round(circleSize), 10, 40);
}

How map() Works

The map() function takes 5 parameters:

map(value, start1, stop1, start2, stop2)
  • value - The number you want to convert
  • start1 - The minimum of the current range
  • stop1 - The maximum of the current range
  • start2 - The minimum of the target range
  • stop2 - The maximum of the target range

So map(mouseX, 0, width, 10, 100) means:

  • Take mouseX
  • Which goes from 0 to width
  • And convert it to a number from 10 to 100

Multiple Mappings!

Let's control multiple things with the mouse position:

function setup() {
  createCanvas(400, 400);
}

function draw() {
  // Map mouseX to background color
  let bgColor = map(mouseX, 0, width, 0, 255);
  background(bgColor);
  
  // Map mouseY to circle color
  let circleColor = map(mouseY, 0, height, 0, 255);
  fill(circleColor, 0, 255 - circleColor);
  
  // Map mouseX to circle size
  let size = map(mouseX, 0, width, 20, 200);
  
  // Map mouseY to circle position
  let circleY = map(mouseY, 0, height, 100, 300);
  
  circle(200, circleY, size);
}

Move your mouse around and watch how everything changes! The background, color, size, and position all respond to your mouse!

Reverse Mapping

You can even reverse the mapping:

function setup() {
  createCanvas(400, 400);
}

function draw() {
  background(220);
  
  // Normal mapping: left = small, right = big
  let size1 = map(mouseX, 0, width, 10, 100);
  
  // Reverse mapping: left = big, right = small
  let size2 = map(mouseX, 0, width, 100, 10);
  
  fill(255, 0, 0);
  circle(200, 150, size1);
  
  fill(0, 0, 255);
  circle(200, 250, size2);
}

2.6 createGraphics() - Drawing Layers

This is a bit advanced, but super cool! createGraphics() lets you create a separate drawing layer - like having multiple transparent sheets of paper you can draw on separately!

let extraCanvas;

function setup() {
  createCanvas(400, 400);
  extraCanvas = createGraphics(400, 400);
  extraCanvas.clear();  // Make it transparent
  background(220);
}

function draw() {
  // Draw on the main canvas
  background(220);
  fill(0, 0, 255);
  circle(mouseX, mouseY, 50);
  
  // Draw the extra canvas on top
  image(extraCanvas, 0, 0);
  
  // Draw on the extra canvas when mouse is pressed
  if (mouseIsPressed) {
    extraCanvas.fill(255, 0, 0);
    extraCanvas.noStroke();
    extraCanvas.circle(mouseX, mouseY, 20);
  }
}

This creates a painting program where the blue circle follows your mouse, but the red paint stays on screen!

Now it's your turn! Try these exercises to practice using variables:

Exercise 2.1: Bouncing Ball

Create a ball that moves horizontally across the screen. When it reaches the edge, make it "teleport" back to the start. Requirements:

  • Use a variable for the x position
  • Make the ball move by incrementing the variable
  • Reset the position when it goes off screen
  • Bonus: Add a variable for speed!

Create a circle that grows and shrinks continuously:

  • Use a variable for the size
  • Use another variable for the growth amount
  • When the circle gets too big OR too small, reverse the growth
  • Hint: You'll need to think about when to make growth positive or negative

Exercise 2.3: Random Colors

Create a sketch where:

  • The background color changes randomly every time you click
  • Circles appear at random positions with random colors
  • Use variables to store the background color so it doesn't change every frame

Create a circle that follows the mouse but with a delay:

  • Use variables for the circle's position
  • Instead of setting them directly to mouseX/mouseY, move them gradually toward the mouse
  • Hint: circleX = circleX + (mouseX - circleX) * 0.05

  • Map mouseX to the red value (0-255)

  • Use random colors
  • Bonus: Make them fade out over time!

2.8 Common Mistakes and Debugging Tips

Mistake 1: Forgetting to Declare Variables

// Wrong - no let keyword
x = 100;

// Right
let x = 100;

Mistake 2: Re-declaring Variables

let x = 100;

function draw() {
  let x = 200;  // This creates a NEW variable!
}

// Better:
let x = 100;

function draw() {
  x = 200;  // This updates the existing variable
}

Mistake 3: Variable Scope Issues

function setup() {
  let x = 100;  // Only exists in setup()
}

function draw() {
  circle(x, 200, 50);  // Error! x doesn't exist here
}

Debugging with console.log()

When your variables aren't doing what you expect, use console.log() to see their values:

let x = 0;
let speed = 5;

function draw() {
  background(220);
  circle(x, 200, 50);
  
  x += speed;
  
  // Debug: see what's happening
  console.log("x is: " + x);
  console.log("speed is: " + speed);
}
