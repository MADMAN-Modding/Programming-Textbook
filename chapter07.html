Chapter 7

Arrays

Arrays of objects, mouse interaction, and splice() with backward loops.

We made it! Final chapter. In Chapter 6 we built a Ball class but ran into a wall: making 50 balls means 50 variable names. Painful. Arrays are the answer. An array is a list — one variable that holds many values in order. Combine arrays with classes and you've unlocked the entire creative-coding universe. Let's do this.

What is an array? #

An array is a numbered list. You make one with square brackets:

let scores = [88, 92, 75, 100, 61];
let colors = ["red", "blue", "green"];

Each value gets a number called an index, starting at zero:

IndexValue
scores[0]88
scores[1]92
scores[2]75
scores[3]100
scores[4]61
Arrays start at 0 The first item is array[0], not array[1]. The last index is array.length - 1. This is the source of a lot of off-by-one bugs. Get used to it; it's everywhere in programming.

You can ask an array how many items it holds:

scores.length     // 5

Coding Train 7.1 — Arrays in JavaScript

Looping over an array #

The whole point of arrays is doing the same thing to every item. That means loops. Here's the most common pattern in all of JavaScript:

for (let i = 0; i < scores.length; i++) {
  print(scores[i]);
}

The counter i starts at 0, goes up to (but not including) scores.length, and at each step we use scores[i] to grab the current value.

The for...of shortcut For when you don't need the index, you can write for (let s of scores) { print(s); }. Cleaner. But the indexed version is more flexible — we'll need it shortly.

Arrays of objects — the killer combo #

Remember our three-ball mess from Chapter 6? Watch what arrays do for it:

let balls = [];

function setup() {
  createCanvas(400, 400);
  for (let i = 0; i < 50; i++) {
    balls.push(new Ball(random(width), random(height), random(10, 40)));
  }
}

function draw() {
  background(220);
  for (let i = 0; i < balls.length; i++) {
    balls[i].move();
    balls[i].bounce();
    balls[i].show();
  }
}

50 balls. Same code, no extra typing. Want 500? Change one number. Want 5,000? You get the idea.

That balls.push(...) line adds a new item to the end of the array. push is one of several useful array methods:

MethodWhat it does
arr.push(item)Add to the end
arr.pop()Remove from the end
arr.splice(i, n)Remove n items starting at index i
arr.lengthHow many items

Mouse interaction: spawn on click #

Here's where it gets dynamic. Let's start with no balls, and add a new one wherever the user clicks:

let balls = [];

function setup() {
  createCanvas(400, 400);
}

function draw() {
  background(220);
  for (let i = 0; i < balls.length; i++) {
    balls[i].move();
    balls[i].bounce();
    balls[i].show();
  }
}

function mousePressed() {
  balls.push(new Ball(mouseX, mouseY, random(10, 40)));
}

Click around. Each click adds a new ball that bounces around with all the others. This pattern — array of objects + spawn on event — powers particle systems, bullets in a game, snowflakes, fireworks, you name it.

Coding Train 7.3 — Arrays of Objects

Removing items: splice() and the backward loop trick #

Adding things is easy. Removing things is where people get burned. Let's say each ball has a this.life property that ticks down every frame, and we want to remove it when it reaches zero.

You might try this — but it has a bug:

// BUGGY — don't do this
for (let i = 0; i < balls.length; i++) {
  balls[i].update();
  if (balls[i].life <= 0) {
    balls.splice(i, 1);
  }
}

Here's the problem: when you splice out item at index 3, everything at index 4 and beyond slides down by one. The thing that used to be at index 4 is now at index 3. But your loop already moved i to 4, so you skip an item. Bugs.

The fix is gorgeous in its simplicity: loop backward.

// CORRECT — backward loop
for (let i = balls.length - 1; i >= 0; i--) {
  balls[i].update();
  if (balls[i].life <= 0) {
    balls.splice(i, 1);
  }
}

When you remove item 7 while going backward, the next item you'll check is item 6, which was always going to be item 6. Removing from the end doesn't affect the indices behind you. Bug gone.

Coding Train 7.4 — Removing Objects from Arrays

Memorize this pattern for (let i = arr.length - 1; i >= 0; i--) — anytime you might delete items from the array while iterating, loop backward. Tattoo it on your brain.

Putting it all together: a particle system #

Here's a sketch that combines everything from this book. Spawn particles wherever the mouse moves; each particle has a life; when its life runs out, it gets removed.

let particles = [];

class Particle {
  constructor(x, y) {
    this.x = x;
    this.y = y;
    this.xspeed = random(-2, 2);
    this.yspeed = random(-2, 2);
    this.life = 255;
  }
  update() {
    this.x += this.xspeed;
    this.y += this.yspeed;
    this.life -= 3;
  }
  show() {
    noStroke();
    fill(237, 34, 93, this.life);
    ellipse(this.x, this.y, 10, 10);
  }
}

function setup() {
  createCanvas(600, 400);
}

function draw() {
  background(20);
  particles.push(new Particle(mouseX, mouseY));

  for (let i = particles.length - 1; i >= 0; i--) {
    particles[i].update();
    particles[i].show();
    if (particles[i].life <= 0) {
      particles.splice(i, 1);
    }
  }
}

That's a real-deal particle system. Trails of pink dots follow your mouse and fade away. Beautiful. And you wrote it.

Common Beginner Mistakes #

Off-by-one with indexes A 5-item array has indexes 0 through 4. scores[5] doesn't exist — you'll get undefined.
Forward loop + splice We just covered this, but it bears repeating. Modifying an array while iterating forward will skip items. Backward loop, every time.
Forgetting new inside push balls.push(Ball(100, 100, 40)); — missing new, you push undefined into the array. Always balls.push(new Ball(...));.
Calling .length() instead of .length length is a property, not a method. No parentheses. balls.length, not balls.length().
