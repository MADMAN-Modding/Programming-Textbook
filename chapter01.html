Chapter 1

Getting Started

Canvas, shapes, color, and the rhythm of setup() and draw().

Alright, alright, alright! Welcome to p5.js. If you've never written a line of code in your life, or if you've done some and want a friendlier way back in — you are in exactly the right place. p5.js is a JavaScript library for creative coding, and by the end of this chapter you'll have drawn shapes, picked colors, and made your very first animation. Let's go!

What is p5.js? #

p5.js is the JavaScript reincarnation of Processing, a language built for artists, designers, students, and educators to make pictures, animations, and interactive things with code. Instead of building a calculator or a database, you build a little world on a digital canvas — and you watch it come alive.

You don't need to install anything. Head to the p5.js Web Editor and you can start coding immediately, right in your browser.

Coding Train 1.1 — Introduction to p5.js

The Canvas #

Everything in p5.js happens on a canvas, which is just a rectangle of pixels. You create one inside a special function called setup(), which runs once when the program starts.

function setup() {
  createCanvas(400, 400);
}

That's it! That gives you a 400-pixel wide, 400-pixel tall canvas. You'll see a blank rectangle appear when you run the sketch.

Coordinates: not what your math teacher taught you

Here's the first thing that will surprise you. In math class, (0, 0) sits in the middle of the page and y goes up. In p5.js, (0, 0) is in the top-left corner of the canvas, and y goes down. Larger y means lower on the screen.

// On a 400x400 canvas:
//   top-left     = (0, 0)
//   top-right    = (400, 0)
//   bottom-left  = (0, 400)
//   bottom-right = (400, 400)
//   center       = (200, 200)
Heads up This flipped y-axis trips up everybody at first. When you mean "higher on screen," type a smaller y.

Drawing Shapes #

Once you have a canvas, you can draw on it. Here are the basics:

point(x, y);
line(x1, y1, x2, y2);
rect(x, y, width, height);
ellipse(x, y, width, height);  // circle if width == height
triangle(x1, y1, x2, y2, x3, y3);

Each one takes some numbers and draws something on the canvas. Let's put a circle right in the middle of a 400×400 canvas:

function setup() {
  createCanvas(400, 400);
  ellipse(200, 200, 100, 100);
}
Where do shapes anchor? By default, rect() anchors at its top-left corner, but ellipse() anchors at its center. You can change rectangles to center-anchor with rectMode(CENTER).

Color #

By default the canvas is gray and everything you draw is white with a thin black outline. You can change that with three friendly functions:

FunctionWhat it does
background(r, g, b)Fills the entire canvas with a color
fill(r, g, b)Sets the inside color of shapes drawn after it
stroke(r, g, b)Sets the outline color of shapes drawn after it

Colors are RGB: three numbers from 0 to 255 for red, green, and blue. You can also pass one number for a grayscale value, or use noFill() and noStroke() to turn them off entirely.

Coding Train 1.3 — Shapes & drawing

function setup() {
  createCanvas(400, 400);
  background(20, 20, 40);     // dark navy
  stroke(255);                 // white outline
  strokeWeight(3);
  fill(237, 34, 93);          // p5 pink
  ellipse(200, 200, 150, 150);
}
Order matters p5 paints shapes in the order you call them, like layers of paint. A background() call erases everything underneath it — so call it first, not last!

setup() vs draw() #

Static pictures are great, but the real fun starts with animation. p5.js gives you a second special function called draw(). Here's the deal:

  • setup() runs once, at the very start.
  • draw() runs over and over, about 60 times per second.

That's the heartbeat. Every line of code inside draw() runs again every frame. That's what makes things move.

Coding Train 1.4 — Color

function setup() {
  createCanvas(400, 400);
}

function draw() {
  background(220);             // runs every frame!
  ellipse(200, 200, 80, 80);
}

Right now this looks identical to a static sketch — but trust me, this template is the doorway to everything cool. In the next chapter we'll use it to make things move.

Common Beginner Mistakes #

Forgetting the parentheses Every function call needs (), even if there's nothing inside. noFill is wrong. noFill() is right.
Mixing up rect and ellipse anchors rect(100, 100, 50, 50) draws a rectangle whose top-left is at (100, 100). ellipse(100, 100, 50, 50) draws an ellipse whose center is at (100, 100). If a circle isn't where you expect, this is usually why.
Capital letters matter JavaScript is case-sensitive. createCanvas works; CreateCanvas or createcanvas will not.
Putting background() at the bottom of draw() If background() is the last thing you call, it paints over everything you just drew. Always call it first inside draw().

Practice Exercises #

Exercise 1.1 — Self portrait Draw a face on a 400×400 canvas. Use at least one rectangle, one ellipse, one triangle, and one line. Pick your own colors for the background, the face, and the eyes. Bonus: give it hair.
Exercise 1.2 — Your school's flag Recreate the flag of your school, your country, or your favorite sports team using only basic shapes and colors. No text needed — just shapes.
Exercise 1.3 — Predict the order Before running it, write down on paper what this sketch will look like. Then run it. Were you right? 
function setup() {
  createCanvas(300, 300);
  background(200);
  fill(255, 0, 0);

}
