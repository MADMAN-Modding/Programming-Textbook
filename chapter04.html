Chapter 4

Loops

while, for, nested loops, and beautiful grids.

Imagine I asked you to draw 100 circles in a row, each 10 pixels apart. With what you know now, you'd type ellipse() a hundred times. Don't do that. Loops are how we ask the computer to repeat itself for us. They're one of the most powerful tools in your toolbox. Let's dig in!

while loops #

A while loop repeats some code as long as a condition is true. Read it out loud: "while x is less than 400, do this stuff."

let x = 0;
while (x < 400) {
  ellipse(x, 200, 20, 20);
  x = x + 40;
}

That draws 10 circles in a horizontal line. Let's trace through it:

  1. x starts at 0. Is 0 < 400? Yes. Draw a circle at x=0. Then x becomes 40.
  2. Is 40 < 400? Yes. Draw a circle at x=40. Then x becomes 80.
  3. ...and so on until x is 400. Is 400 < 400? No. Stop.
The infinite loop trap If you forget to change x inside the loop, the condition stays true forever and your browser tab freezes. Always make sure something inside the loop moves you toward the exit condition.

Coding Train 4.1 — while and for loops

for loops #

Looping with a counter is so common that JavaScript gives us a shorter way to write it: the for loop. It packages the three pieces (start, condition, change) into one line.

for (let i = 0; i < 10; i++) {
  print(i);   // prints 0, 1, 2, ..., 9
}

The three pieces inside the parentheses, separated by semicolons:

  1. Start: let i = 0 — declare a counter and give it a starting value.
  2. Condition: i < 10 — keep looping while this is true.
  3. Update: i++ — runs after each iteration. i++ means "add 1 to i."

So our row of circles becomes much tidier:

for (let x = 0; x < width; x += 40) {
  ellipse(x, 200, 20, 20);
}
i, j, k The counter is conventionally called i (for "index") when looping over a count. If you nest loops, the inner ones become j and k. You don't have to use these names — sometimes x or row makes more sense — but you'll see i everywhere.

Loops inside draw() #

Remember that draw() already runs 60 times a second. If you put a loop inside draw(), the whole loop runs every frame. That's how you draw static patterns that interact with the mouse:

function setup() {
  createCanvas(400, 400);
}

function draw() {
  background(220);
  for (let x = 0; x < width; x += 40) {
    ellipse(x, mouseY, 20, 20);
  }
}

A whole row of circles, all moving together with the mouse's y position. So much wow.

Nested loops: hello, grid #

Put a loop inside a loop and you can draw a grid. The outer loop walks down rows; the inner loop walks across each row.

function setup() {
  createCanvas(400, 400);
  background(220);
  for (let y = 20; y < height; y += 40) {
    for (let x = 20; x < width; x += 40) {
      ellipse(x, y, 20, 20);
    }
  }
}

Read that carefully: when y is 20, the inner loop runs all the way across (x = 20, 60, 100, …). Then y becomes 60, and the inner loop runs all the way across again. By the time you're done, you've drawn 100 circles with eight lines of code.

Coding Train 4.2 — Nested loops

It's grids all the way down Nested loops are how you build chess boards, pixel art, fields of grass, snowstorms, kaleidoscopes, and pretty much every cool generative art piece you've ever seen on Twitter. This pattern is gold.

Combining loops with map() #

Here's where it gets really fun. Use the counter i with map() to make a value smoothly change across all the shapes in the loop:

function draw() {
  background(220);
  for (let x = 0; x < width; x += 20) {
    let c = map(x, 0, width, 0, 255);
    fill(c, 100, 255 - c);
    ellipse(x, height/2, 20, 20);
  }
}

The leftmost circle is purplish, the rightmost is reddish, and everything in between fades between them. You've made your first gradient.

Common Beginner Mistakes #

Forgetting to update the counter A while loop with no x = x + 1 inside is the infinite loop of doom. If your browser locks up, this is probably why. Reload the tab and check.
Off-by-one for (let i = 0; i <= 10; i++) runs 11 times (0 through 10). for (let i = 0; i < 10; i++) runs 10 times. Use < when you want N iterations starting at 0.
Using the same counter name in nested loops for (let i = ...; for (let i = ...)) — both counters share the name and they fight. Use i outside and j inside.
Putting setup() code inside draw() (or vice versa) If you put your loop inside setup(), it runs once and the grid is permanent. If you put it inside draw(), it runs every frame and can react to the mouse. Both are useful — just know which one you want.

Practice Exercises #

Exercise 4.1 — Stripes Draw 20 vertical stripes across the canvas. Alternate their colors using an if/else inside the loop. Hint: use the % (modulo) operator: if (i % 2 === 0).
Exercise 4.2 — Color grid Make a grid of small squares where each square's color depends on its x and y position. Use map() to mix red based on x and blue based on y.
Exercise 4.3 — Spotlight Make a 20×20 grid of circles. The size of each circle should depend on how close it is to the mouse — circles near the mouse are big; far-away circles are small. Use dist(x, y, mouseX, mouseY).
Exercise 4.4 — Checkerboard Draw an 8×8 checkerboard. Each square should alternate between black and white. Hint: if ((row + col) % 2 === 0).
