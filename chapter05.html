Chapter 5

Functions

Parameters, return values, and the magic of scope.

You've already been using functions this whole book! setup(), draw(), ellipse(), fill() — those are all functions. Some came with p5.js, some you wrote yourself. In this chapter we're going to learn to write our own custom functions, give them parameters, and have them return values. This is where your code starts feeling like your code.

Why bother writing functions? #

Three big reasons. Functions let you:

  1. Stop repeating yourself. Write the code once, call it as many times as you want.
  2. Hide complexity. "Draw a face" is easier to think about than 20 lines of ellipse() and rect() calls.
  3. Reuse code in different places. Once you have a drawTree() function, you can drop trees anywhere on your canvas.

Coding Train 5.1 — Functions in p5.js

Defining a function #

You already know the shape, because you've been writing setup() and draw() all along:

function drawFace() {
  ellipse(200, 200, 100, 100);   // head
  ellipse(180, 190, 10, 10);     // left eye
  ellipse(220, 190, 10, 10);     // right eye
  line(180, 220, 220, 220);     // mouth
}

That's a recipe. But the recipe doesn't run until you call it:

function setup() {
  createCanvas(400, 400);
  drawFace();   // here's the call!
}

The parentheses are how you say "run this now." Without them, you're just pointing at the recipe.

Parameters: making functions flexible #

Our drawFace() draws a face in exactly one spot. Boring. Let's let the caller decide where the face goes by adding parameters:

function drawFace(x, y) {
  ellipse(x,      y,      100, 100);
  ellipse(x - 20, y - 10, 10,  10);
  ellipse(x + 20, y - 10, 10,  10);
  line(x - 20, y + 20, x + 20, y + 20);
}

Now I can plant a face anywhere:

function setup() {
  createCanvas(600, 400);
  drawFace(150, 200);
  drawFace(450, 200);
}

The parameters (x, y) are the names you use inside the function — variables that get filled in with the arguments (150, 200) you pass in. Same idea as ellipse(x, y, w, h): somebody who built p5 wrote a function with four parameters, and you keep filling them in.

Add as many parameters as you need drawFace(x, y, size, eyeColor) — you can add as many parameters as makes sense for what your function needs to know.

Coding Train 5.2 — Function parameters

Return values #

So far, our functions do things (draw shapes). Sometimes you want a function that computes and gives back a value. That's what return is for.

function average(a, b) {
  return (a + b) / 2;
}

let midX = average(0, width);   // midX is now 200 on a 400-wide canvas

You've been doing this with built-in functions all along! random(100) doesn't do anything visible; it gives you back a number. map(...) too. Same pattern.

function distance2D(x1, y1, x2, y2) {
  let dx = x2 - x1;
  let dy = y2 - y1;
  return sqrt(dx*dx + dy*dy);
}

(p5 already has dist() for this — we're just showing how it might look inside.)

return exits the function The moment return runs, the function is done. Anything after it doesn't execute. You can use that to bail out early if something's wrong.

Scope: where variables live #

Every variable has a scope — a region of the program where it exists. Variables declared with let or const inside curly braces are local: they only exist inside those braces.

let globalCount = 0;     // global - visible everywhere

function draw() {
  let r = random(255);   // local - only exists inside draw()
  fill(r);
  ellipse(200, 200, 50, 50);
  globalCount = globalCount + 1;
}

function mousePressed() {
  print(globalCount);    // works! global
  print(r);             // ERROR! r doesn't exist here
}

Why does scope exist? So you don't accidentally clobber things. A counter named i in one function should have nothing to do with a counter named i in another function. Scope keeps them separate.

Coding Train 5.3 — Scope

Rule of thumb Keep variables as local as you can. If only one function needs it, declare it inside that function. Only make a variable global when many functions truly need to share it (like the position of a bouncing ball that's read in draw() and updated in mousePressed()).

Putting it together #

Let's build something. A function that draws a tree, with parameters for position and size:

function setup() {
  createCanvas(600, 400);
  background(200, 230, 255);
  // a little forest
  for (let i = 0; i < 8; i++) {
    drawTree(random(width), 350, random(40, 90));
  }
}

function drawTree(x, y, size) {
  // trunk
  fill(90, 60, 30);
  noStroke();
  rect(x - size/10, y - size, size/5, size);
  // leaves
  fill(40, 140, 60);
  ellipse(x, y - size, size, size);
}

Eight random trees, fifteen lines of code. That's the power of functions.

Common Beginner Mistakes #

Defining a function but never calling it Writing function drawFace() { ... } doesn't draw anything — it just defines the recipe. You also have to call drawFace(); somewhere.
Forgetting return A function that's supposed to give back a value but has no return statement gives back undefined. If you do math with it, you'll get NaN (Not a Number). Always return.
Using a local variable outside its function A variable declared inside one function doesn't exist in another. If you need it everywhere, declare it at the top of the sketch (global), or pass it in as a parameter.
Naming a custom function the same as a p5 function If you write function draw() { ... } twice, you've overridden the first one. If you accidentally write function ellipse() { ... }, you've broken p5's built-in ellipse. Pick distinctive names.
