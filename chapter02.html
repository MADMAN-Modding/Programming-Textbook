Chapter 2

Variables

mouseX, mouseY, let, animation, random(), and map().

Welcome back! In Chapter 1 we drew shapes, but the canvas just sat there staring at us. In this chapter we're going to make things move. The secret ingredient? Variables. Let's dig in!

What is a variable? #

A variable is a name you give to a piece of information so you can use it later. Think of it like a labeled jar: you put a value inside, you slap a label on it, and from then on you can ask for the contents by name.

let circleSize = 50;
ellipse(200, 200, circleSize, circleSize);

The keyword let tells JavaScript "I'm making a new variable." After this line, circleSize means 50 everywhere in your sketch. Change it once at the top, and every shape that uses it changes too. That alone is a superpower.

Coding Train 2.1 — Variables in p5.js (mouseX, mouseY)

mouseX and mouseY #

p5.js gives you a couple of variables for free — they already exist, you don't have to declare them, and they update automatically. The two most useful ones are mouseX and mouseY: at any moment, they hold the current x and y position of the mouse on the canvas.

function setup() {
  createCanvas(400, 400);
}

function draw() {
  background(220);
  ellipse(mouseX, mouseY, 50, 50);
}

Run that sketch and move your mouse. A circle follows you around. Magic. But it's not magic — every frame, draw() runs again, and mouseX/mouseY have new values, so the circle gets drawn in a new spot.

Try this experiment Comment out the background(220); line by adding // in front of it. Now move your mouse. What happens? Why?

Declaring your own variables #

You're not stuck with the built-in ones — you can make your own. Use let when you plan to change the value later. Use const when the value should never change.

let x = 50;       // can change
const SIZE = 80;   // cannot change after this

If you try to reassign SIZE later, JavaScript will yell at you. That's a feature, not a bug — it stops you from accidentally clobbering something important.

Coding Train 2.2 — The map() function

Animation: changing a variable over time #

Here's the recipe for animation in p5.js. It only has three ingredients:

  1. Declare a variable outside of setup() and draw().
  2. Use that variable inside draw() to position a shape.
  3. Change the variable a little bit at the end of draw().
let x = 0;

function setup() {
  createCanvas(400, 400);
}

function draw() {
  background(220);
  ellipse(x, 200, 50, 50);
  x = x + 2;   // every frame, move 2 pixels right
}

That circle now slides smoothly across the canvas. The variable changes by 2 every frame, and draw() runs ~60 times per second, so you get 120 pixels per second of motion. Beautiful.

Shorthand x = x + 2; is so common it has a shortcut: x += 2;. Same thing, less typing. Likewise x -= 2, x *= 2, etc.

random() — controlled chaos #

Pictures with a little randomness in them feel alive. The random() function gives you back a random number every time you call it.

random(100);          // random number between 0 and 100
random(50, 100);      // random number between 50 and 100
random(255);          // useful for random colors!

Coding Train 2.3 — random() in p5.js

Here's a sketch that paints random colorful dots forever. Notice we don't call background() inside draw() — so the dots accumulate:

function setup() {
  createCanvas(400, 400);
  background(0);
  noStroke();
}

function draw() {
  fill(random(255), random(255), random(255), 150);
  ellipse(random(width), random(height), 20, 20);
}
Free variables: width and height width and height are two more built-in variables. They always equal whatever you passed to createCanvas(). Use them instead of hardcoding numbers — your sketch will resize cleanly later.

map() — translate one range to another #

This is one of the most useful functions in all of p5.js. map() takes a number from one range and proportionally squishes or stretches it into a different range.

map(value, fromLow, fromHigh, toLow, toHigh);

Say the mouse moves from x=0 to x=400, and you want to use it to control a circle's size from 10 to 200. map() handles that in one line:

function draw() {
  background(220);
  let diameter = map(mouseX, 0, width, 10, 200);
  ellipse(200, 200, diameter, diameter);
}

Coding Train 2.4 — map() continued

Anywhere you find yourself thinking "as this goes from A to B, I want that to go from C to D" — that's a map().

Coding Train 2.5 — More on variables & a peek at conditionals

Common Beginner Mistakes #

Declaring variables inside draw() If you write let x = 0; inside draw(), the variable gets reset to 0 every single frame. It will never move. Declare it above setup() instead, and only modify it inside draw().
= versus == A single equals sign assigns a value. Double equals (and triple equals!) compare values. We'll see comparison in the next chapter. Using = when you meant == is a classic bug.
Forgetting that random() picks a new number every call ellipse(random(width), random(height), random(50), random(50)); — the two random(50)s give you different numbers, so your circle is squashed. If you want both the same, store it in a variable first.
Using a const where you needed a let If you ever see an error like "Assignment to constant variable", you used const for a value that needs to change. Switch it to let.
